1. Jelaskan perbedaan antara pengembangan aplikasi mobile secara hybrid (contoh: Flutter) dan pengembangan aplikasi mobile secara native (contoh: Kotlin untuk pengembangan aplikasi Android)
Jawaban :
Pengembangan Aplikasi Mobile Secara Hybrid:

Pendekatan: Pengembangan aplikasi mobile secara hybrid melibatkan penggunaan satu kode basis yang dapat dijalankan pada berbagai platform, seperti Android dan iOS. Framework hybrid menyediakan lapisan abstraksi yang mengizinkan pengembang untuk membuat satu set kode yang dapat digunakan kembali pada kedua platform.

Bahasa Pemrograman dan Framework: Contoh framework hybrid yang populer adalah Flutter (menggunakan bahasa Dart) dan React Native (menggunakan JavaScript).

Kelebihan:

Pengembangan lebih cepat karena menggunakan satu kode basis.
Lebih efisien dalam hal waktu dan sumber daya.
Memungkinkan pengembang web untuk mengembangkan aplikasi mobile tanpa harus mempelajari bahasa pemrograman yang berbeda.
Kekurangan:

Kinerja mungkin tidak sebaik aplikasi native karena lapisan abstraksi.
Beberapa fitur mungkin sulit diakses atau harus dibuat khusus untuk setiap platform.
Pengembangan Aplikasi Mobile Secara Native:

Pendekatan: Pengembangan aplikasi mobile secara native melibatkan penggunaan bahasa pemrograman dan alat yang ditentukan oleh platform masing-masing, seperti Java atau Kotlin untuk Android, dan Swift atau Objective-C untuk iOS.

Kelebihan:

Kinerja yang lebih baik karena menggunakan fitur dan API platform secara langsung.
Akses penuh ke semua fitur dan fungsionalitas perangkat.
Antarmuka yang lebih halus dan konsisten dengan tata letak platform.
Kekurangan:

Memerlukan pengembangan terpisah untuk setiap platform.
Lebih kompleks dan memerlukan waktu lebih lama.
Memerlukan pengetahuan tentang bahasa pemrograman dan alat yang berbeda.

2. Sebutkan contoh kasus yang tepat untuk pengembangan aplikasi mobile secara hybrid.
Contoh Kasus Tepat untuk Pengembangan Aplikasi Mobile Secara Hybrid:

Pengembangan aplikasi mobile secara hybrid cocok untuk kasus di mana:

Waktu pengembangan menjadi faktor kritis.
Anda ingin menghemat sumber daya dan anggaran dengan menggunakan satu kode basis.
Aplikasi tidak memerlukan fitur yang sangat mendalam yang hanya dapat dicapai dengan akses langsung ke fitur perangkat.
Anda adalah pengembang web yang ingin memperluas ke hadirat mobile tanpa harus belajar bahasa pemrograman baru.
